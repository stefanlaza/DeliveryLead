Scheduled Batch
Pipeline

EventBridge triggers Step Functions on a cron schedule. Lambda validates inputs, AWS Batch runs parallel Fargate jobs, results land in S3, and SNS sends completion alerts. Includes DLQ handling and retry logic.

EventBridge Step Functions AWS Batch Fargate Lambda SQS DLQ S3 SNS
Overview

Pipeline Flow

The batch pipeline runs nightly at 2am UTC. EventBridge fires the trigger, Step Functions orchestrates the workflow, and AWS Batch runs the actual compute jobs in parallel on Fargate — no servers to manage.

EventBridge

Cron trigger at 02:00 UTC. Fires Step Functions execution with date parameters.

Step Functions

Orchestrates validate → parallel batch → notify. Handles retries and failure routing.

Lambda Validator

Checks prerequisites — source data available, no running jobs, sufficient capacity.

AWS Batch

Parallel Fargate jobs. Each job processes one data partition independently.

S3

Job results written to partitioned S3 path. Triggers downstream processing.

SNS

Success and failure notifications to ops team. Links to CloudWatch logs.

Build Guide

01

EventBridge Scheduler

Create a cron rule that fires every night at 2am UTC. Pass the execution date as an input parameter so the pipeline knows which day's data to process.

# EventBridge rule — nightly at 02:00 UTC
aws events put-rule \
  --name nightly-batch-trigger \
  --schedule-expression "cron(0 2 * * ? *)" \
  --state ENABLED \
  --description "Triggers nightly batch pipeline"

# Target — Step Functions state machine
aws events put-targets \
  --rule nightly-batch-trigger \
  --targets '[{
    "Id": "BatchPipelineTarget",
    "Arn": "arn:aws:states:us-east-1:xxx:stateMachine:NightlyBatch",
    "RoleArn": "arn:aws:iam::xxx:role/EventBridgeStepFunctionsRole",
    "Input": "{\"run_date\": \"\"}"
  }]'
02

Lambda Validator

Before launching expensive Batch jobs, the validator checks: source data exists in S3, no other execution is currently running, and the target output path is empty.

import boto3
import json
from datetime import datetime, timedelta

s3 = boto3.client('s3')
sfn = boto3.client('stepfunctions')

def handler(event, context):
    run_date = event.get('run_date', datetime.utcnow().strftime('%Y-%m-%d'))

    # Check 1: Source data exists
    source_key = f"raw/orders/date={run_date}/"
    resp = s3.list_objects_v2(Bucket='prod-datalake-bronze', Prefix=source_key, MaxKeys=1)
    if resp['KeyCount'] == 0:
        raise Exception(f"No source data found for {run_date} at {source_key}")

    # Check 2: No running executions
    running = sfn.list_executions(
        stateMachineArn=event['state_machine_arn'],
        statusFilter='RUNNING'
    )
    if len(running['executions']) > 1:  # >1 because current execution counts
        raise Exception(f"Another execution is already running. Aborting.")

    # Check 3: Output not already processed
    output_key = f"processed/orders/date={run_date}/"
    existing = s3.list_objects_v2(Bucket='prod-datalake-silver', Prefix=output_key, MaxKeys=1)
    if existing['KeyCount'] > 0:
        print(f"Warning: Output already exists for {run_date}. Will overwrite.")

    return {
        'run_date': run_date,
        'source_prefix': source_key,
        'output_prefix': output_key,
        'status': 'validated'
    }
03

AWS Batch — Job Definition

Define the Batch job that runs on Fargate. The container image contains the processing logic — no need to manage EC2 instances or AMIs.

# Create compute environment (Fargate)
aws batch create-compute-environment \
  --compute-environment-name prod-batch-fargate \
  --type MANAGED \
  --state ENABLED \
  --compute-resources '{
    "type": "FARGATE",
    "maxvCpus": 256,
    "subnets": ["subnet-private-1a", "subnet-private-1b"],
    "securityGroupIds": ["sg-batch-id"]
  }'

# Job queue
aws batch create-job-queue \
  --job-queue-name prod-batch-queue \
  --state ENABLED \
  --priority 100 \
  --compute-environment-order '[{"order": 1, "computeEnvironment": "prod-batch-fargate"}]'

# Job definition
aws batch register-job-definition \
  --job-definition-name process-orders \
  --type container \
  --platform-capabilities FARGATE \
  --container-properties '{
    "image": "123456789.dkr.ecr.us-east-1.amazonaws.com/batch-processor:latest",
    "resourceRequirements": [
      {"type": "VCPU", "value": "2"},
      {"type": "MEMORY", "value": "4096"}
    ],
    "jobRoleArn": "arn:aws:iam::xxx:role/BatchJobRole",
    "executionRoleArn": "arn:aws:iam::xxx:role/BatchExecutionRole",
    "environment": [
      {"name": "OUTPUT_BUCKET", "value": "prod-datalake-silver"}
    ],
    "logConfiguration": {"logDriver": "awslogs"}
  }'
04

Step Functions — Parallel Execution

Use the Map state to run Batch jobs in parallel — one job per data partition. This reduces a 4-hour sequential run to a 30-minute parallel run.

{
  "Comment": "Nightly Batch Pipeline",
  "StartAt": "Validate",
  "States": {
    "Validate": {
      "Type": "Task",
      "Resource": "arn:aws:states:::lambda:invoke",
      "Parameters": {
        "FunctionName": "batch-validator",
        "Payload.$": "$"
      },
      "ResultPath": "$.validation",
      "Next": "GetPartitions"
    },
    "GetPartitions": {
      "Type": "Task",
      "Resource": "arn:aws:states:::lambda:invoke",
      "Parameters": {"FunctionName": "get-partitions", "Payload.$": "$"},
      "ResultPath": "$.partitions",
      "Next": "ProcessInParallel"
    },
    "ProcessInParallel": {
      "Type": "Map",
      "ItemsPath": "$.partitions.Payload",
      "MaxConcurrency": 10,
      "Iterator": {
        "StartAt": "RunBatchJob",
        "States": {
          "RunBatchJob": {
            "Type": "Task",
            "Resource": "arn:aws:states:::batch:submitJob.sync",
            "Parameters": {
              "JobName.$": "States.Format('process-{}', $.partition_id)",
              "JobDefinition": "process-orders",
              "JobQueue": "prod-batch-queue",
              "ContainerOverrides": {
                "Environment": [
                  {"Name": "PARTITION_ID", "Value.$": "$.partition_id"},
                  {"Name": "RUN_DATE", "Value.$": "$.run_date"}
                ]
              }
            },
            "Retry": [{"ErrorEquals": ["States.ALL"], "MaxAttempts": 2, "IntervalSeconds": 120}],
            "End": true
          }
        }
      },
      "Next": "NotifySuccess",
      "Catch": [{"ErrorEquals": ["States.ALL"], "Next": "SendToDeadLetter"}]
    },
    "SendToDeadLetter": {
      "Type": "Task",
      "Resource": "arn:aws:states:::sqs:sendMessage",
      "Parameters": {
        "QueueUrl": "https://sqs.us-east-1.amazonaws.com/xxx/batch-dlq",
        "MessageBody.$": "States.JsonToString($)"
      },
      "Next": "NotifyFailure"
    },
    "NotifySuccess": {
      "Type": "Task",
      "Resource": "arn:aws:states:::sns:publish",
      "Parameters": {
        "TopicArn": "arn:aws:sns:us-east-1:xxx:batch-alerts",
        "Subject": "Batch Pipeline Completed",
        "Message.$": "States.Format('Nightly batch completed for {}. All partitions processed.', $.validation.Payload.run_date)"
      },
      "End": true
    },
    "NotifyFailure": {
      "Type": "Task",
      "Resource": "arn:aws:states:::sns:publish",
      "Parameters": {
        "TopicArn": "arn:aws:sns:us-east-1:xxx:batch-alerts",
        "Subject": "ALERT: Batch Pipeline Failed",
        "Message.$": "States.JsonToString($)"
      },
      "End": true
    }
  }
}
Performance Note

MaxConcurrency: 10 runs 10 partitions simultaneously. Increase this for larger datasets — Fargate scales automatically. Setting it to 0 means unlimited concurrency.

05

Dead Letter Queue — Failure Handling

Failed job payloads are sent to an SQS DLQ. A separate Lambda processes the DLQ and determines whether to retry, alert, or discard.

# Create DLQ
aws sqs create-queue \
  --queue-name batch-pipeline-dlq \
  --attributes '{
    "MessageRetentionPeriod": "1209600",
    "VisibilityTimeout": "300"
  }'

# Lambda to process DLQ
def dlq_handler(event, context):
    for record in event['Records']:
        payload = json.loads(record['body'])
        run_date = payload.get('run_date', 'unknown')
        error = payload.get('error', 'unknown error')

        # Log to CloudWatch with structured data
        print(json.dumps({
            'level': 'ERROR',
            'event': 'batch_job_failed',
            'run_date': run_date,
            'error': error,
            'retry_count': record['attributes']['ApproximateReceiveCount']
        }))

        # After 3 retries, create a PagerDuty/Slack alert
        if int(record['attributes']['ApproximateReceiveCount']) >= 3:
            send_pagerduty_alert(run_date, error)
