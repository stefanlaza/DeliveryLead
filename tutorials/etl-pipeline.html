ETL Pipeline
Architecture

A production ETL pipeline ingesting data from REST APIs, databases, and flat files — transforming with AWS Glue PySpark — loading into Redshift. Includes error handling, data quality checks, and monitoring.

AWS Glue PySpark S3 Redshift Lambda Step Functions CloudWatch SNS
Overview

Pipeline Architecture

The pipeline follows a classic medallion architecture — raw data lands in a Bronze S3 layer, Glue transforms it to Silver, and final aggregations produce Gold data ready for analytics in Redshift.

Ingestion

Lambda pulls from REST APIs, JDBC sources, and S3 file drops into Bronze layer.

Transform

Glue PySpark jobs clean, validate, deduplicate, and enrich data into Silver layer.

Aggregate

Glue jobs compute business metrics and KPIs into Gold layer tables.

Load

Redshift COPY command loads Gold data. BI tools query Redshift directly.

Build Guide

01

S3 Bucket Structure — Medallion Layers

Create three S3 buckets or prefixes for the Bronze, Silver, and Gold layers. Use separate buckets per environment and enable versioning on all layers.

# Create S3 buckets for each layer
aws s3api create-bucket --bucket prod-datalake-bronze --region us-east-1
aws s3api create-bucket --bucket prod-datalake-silver --region us-east-1
aws s3api create-bucket --bucket prod-datalake-gold --region us-east-1

# Enable versioning — important for reprocessing
aws s3api put-bucket-versioning \
  --bucket prod-datalake-bronze \
  --versioning-configuration Status=Enabled

# Lifecycle policy — move bronze to Glacier after 90 days
aws s3api put-bucket-lifecycle-configuration \
  --bucket prod-datalake-bronze \
  --lifecycle-configuration '{
    "Rules": [{
      "Id": "archive-raw",
      "Status": "Enabled",
      "Transitions": [{"Days": 90, "StorageClass": "GLACIER"}],
      "Filter": {"Prefix": ""}
    }]
  }'
02

Lambda Ingestion — REST API Source

Lambda functions run on a schedule to pull data from source systems and land it in the Bronze S3 layer. Each source gets its own Lambda to fail independently.

import boto3
import requests
import json
from datetime import datetime

s3 = boto3.client('s3')
BUCKET = 'prod-datalake-bronze'

def handler(event, context):
    # Build partition path by date
    today = datetime.utcnow()
    prefix = f"api/orders/year={today.year}/month={today.month:02d}/day={today.day:02d}/"

    # Pull from source API with pagination
    page = 1
    records = []
    while True:
        resp = requests.get(
            'https://api.source.com/orders',
            params={'page': page, 'limit': 1000},
            headers={'Authorization': f"Bearer {get_secret('api-key')}"},
            timeout=30
        )
        resp.raise_for_status()
        data = resp.json()
        records.extend(data['items'])

        if not data.get('next_page'):
            break
        page += 1

    # Write as newline-delimited JSON to S3
    filename = f"{prefix}orders_{today.strftime('%H%M%S')}.json"
    body = '\n'.join(json.dumps(r) for r in records)
    s3.put_object(Bucket=BUCKET, Key=filename, Body=body)

    print(f"Ingested {len(records)} records to s3://{BUCKET}/{filename}")
    return {'records': len(records), 'path': filename}
03

AWS Glue Job — Bronze to Silver Transform

The Glue PySpark job reads raw Bronze data, applies data quality checks, cleans nulls, deduplicates, and writes clean Parquet to the Silver layer.

import sys
from awsglue.transforms import *
from awsglue.utils import getResolvedOptions
from pyspark.context import SparkContext
from awsglue.context import GlueContext
from awsglue.job import Job
from pyspark.sql import functions as F
from pyspark.sql.types import TimestampType

args = getResolvedOptions(sys.argv, ['JOB_NAME', 'source_path', 'target_path'])
sc = SparkContext()
glueContext = GlueContext(sc)
spark = glueContext.spark_session
job = Job(glueContext)
job.init(args['JOB_NAME'], args)

# Read Bronze layer
df = spark.read.json(args['source_path'])

print(f"Raw record count: {df.count()}")

# Data quality checks
df_clean = df.filter(
    F.col('order_id').isNotNull() &
    F.col('customer_id').isNotNull() &
    F.col('amount').isNotNull() &
    (F.col('amount') > 0)  # Business rule: no zero-value orders
)

rejected = df.count() - df_clean.count()
print(f"Rejected {rejected} records failing quality checks")

# Standardize and enrich
df_silver = df_clean \
    .withColumn('created_at', F.to_timestamp('created_at')) \
    .withColumn('amount', F.col('amount').cast('decimal(18,2)')) \
    .withColumn('status', F.upper(F.trim(F.col('status')))) \
    .withColumn('year', F.year('created_at')) \
    .withColumn('month', F.month('created_at')) \
    .withColumn('day', F.dayofmonth('created_at')) \
    .dropDuplicates(['order_id'])  # Deduplicate by business key

# Write partitioned Parquet to Silver
df_silver.write \
    .mode('overwrite') \
    .partitionBy('year', 'month', 'day') \
    .parquet(args['target_path'])

print(f"Silver record count: {df_silver.count()}")
job.commit()
04

AWS Glue Job — Silver to Gold Aggregation

The Gold layer job computes business-ready aggregations — daily revenue, order counts by region, customer lifetime value — that BI tools query directly.

from pyspark.sql import functions as F

# Read Silver
orders = spark.read.parquet(args['silver_orders'])
customers = spark.read.parquet(args['silver_customers'])

# Join and compute daily revenue metrics
daily_revenue = orders \
    .join(customers, 'customer_id', 'left') \
    .filter(F.col('status') == 'COMPLETED') \
    .groupBy('year', 'month', 'day', 'region', 'product_category') \
    .agg(
        F.count('order_id').alias('order_count'),
        F.sum('amount').alias('gross_revenue'),
        F.avg('amount').alias('avg_order_value'),
        F.countDistinct('customer_id').alias('unique_customers'),
        F.sum(
            F.when(F.col('is_new_customer'), F.col('amount'))
        ).alias('new_customer_revenue')
    )

# Write to Gold
daily_revenue.write \
    .mode('overwrite') \
    .partitionBy('year', 'month', 'day') \
    .parquet(args['gold_path'])
05

Load to Redshift

Use the Redshift COPY command — it loads in parallel from S3 and is 100x faster than INSERT statements. Always use an IAM role, never credentials in the query.

-- Create target table in Redshift
CREATE TABLE analytics.daily_revenue (
    year         INTEGER,
    month        INTEGER,
    day          INTEGER,
    region       VARCHAR(50),
    product_category VARCHAR(100),
    order_count  INTEGER,
    gross_revenue DECIMAL(18,2),
    avg_order_value DECIMAL(18,2),
    unique_customers INTEGER,
    new_customer_revenue DECIMAL(18,2)
)
DISTKEY(region)
SORTKEY(year, month, day);

-- Load from S3 using COPY
COPY analytics.daily_revenue
FROM 's3://prod-datalake-gold/daily-revenue/'
IAM_ROLE 'arn:aws:iam::123456789:role/RedshiftS3Role'
FORMAT AS PARQUET
SERIALIZETOJSON;

-- Verify load
SELECT COUNT(*), MAX(year || '-' || month || '-' || day)
FROM analytics.daily_revenue;
Performance Tip

Always VACUUM and ANALYZE after bulk loads to update query statistics and reclaim space from deleted rows. Redshift performance degrades significantly without regular maintenance.

Step Functions Orchestration

Use Step Functions to sequence the pipeline stages with error handling, retries, and failure notifications. Each stage only starts if the previous one succeeded.

{
  "Comment": "ETL Pipeline Orchestration",
  "StartAt": "IngestFromAPI",
  "States": {
    "IngestFromAPI": {
      "Type": "Task",
      "Resource": "arn:aws:states:::lambda:invoke",
      "Parameters": {"FunctionName": "etl-ingest-orders"},
      "Retry": [{"ErrorEquals": ["States.ALL"], "MaxAttempts": 3, "IntervalSeconds": 60}],
      "Catch": [{"ErrorEquals": ["States.ALL"], "Next": "NotifyFailure"}],
      "Next": "TransformBronzeToSilver"
    },
    "TransformBronzeToSilver": {
      "Type": "Task",
      "Resource": "arn:aws:states:::glue:startJobRun.sync",
      "Parameters": {"JobName": "bronze-to-silver"},
      "Retry": [{"ErrorEquals": ["States.ALL"], "MaxAttempts": 2}],
      "Catch": [{"ErrorEquals": ["States.ALL"], "Next": "NotifyFailure"}],
      "Next": "AggregateToGold"
    },
    "AggregateToGold": {
      "Type": "Task",
      "Resource": "arn:aws:states:::glue:startJobRun.sync",
      "Parameters": {"JobName": "silver-to-gold"},
